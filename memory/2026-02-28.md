## OpenClaw Discord API 配置问题

### 问题描述
ProjectManager agent 在 Discord 中无法使用，报错 "billing error" 或 "invalid api key"

### 根因
Gateway 读取的是 `agents/xxx/agent/auth-profiles.json` 而不是 `openclaw.json` 中的 API keys！

即使更新了 openclaw.json，如果 auth-profiles.json 存在且包含旧 key，Gateway 会优先使用它。

### 解决方案
1. 删除 `agents/projectmanager/agent/auth-profiles.json`
2. 让 Gateway 使用 openclaw.json 中的配置

### API Keys 状态
- **Minimax**: sk-cp-wnncftw40... (新 key，需确认有效)
- **Kimi**: sk-kimi-nWGyTyn... (已失效，需更换)

### 配置精简
按照用户要求，只保留：
- minimax/MiniMax-M2.5 (主要)
- kimi-coding/k2p5 (备用)
