# 股票投资模型全面优化计划

## 一、当前系统架构诊断

### 现有问题
1. **数据孤岛**：选股、组合、回测三个模块数据未完全打通
2. **缺乏持久化**：用户策略、组合配置无法保存复用
3. **数据质量**：依赖外部数据源，缺乏数据质量校验
4. **用户体验**：操作流程不够流畅，缺少引导
5. **专业度不足**：缺少风控、归因、压力测试等专业功能

---

## 二、分阶段优化计划

### Phase 1: 数据流打通与持久化 (2-3天)

#### 1.1 统一数据状态管理
**现状问题**：
- 三个页面各自维护状态，无法联动
- 页面刷新后数据丢失

**优化方案**：
```
引入 Pinia Store 统一管理：
├── stockScreeningStore (选股状态)
│   ├── selectedFactors
│   ├── screeningResults
│   └── savedStrategies
├── stockPortfolioStore (组合状态)
│   ├── stockPool
│   ├── weights
│   ├── optimizationConfig
│   └── backtestSettings
└── stockBacktestStore (回测状态)
    ├── backtestParams
    ├── results
    └── savedReports
```

**收益**：
- 页面间数据无缝流转
- 刷新后自动恢复状态
- 支持多策略对比

#### 1.2 策略持久化系统
**功能需求**：
- 保存多因子选股策略
- 保存组合优化配置
- 保存回测参数模板
- 策略版本管理

**技术实现**：
```python
# 后端新增表
class ScreeningStrategy(Base):
    id, name, factors, create_time, update_time

class PortfolioConfig(Base):
    id, name, stocks, weights, strategy_type

class BacktestTemplate(Base):
    id, name, params, performance_summary
```

### Phase 3: 组合构建模块增强 (3-4天)

#### 3.1 优化策略增强
**新增策略**：
| 策略 | 说明 | 适用场景 |
|------|------|----------|
| Black-Litterman | 融入主观观点 | 有市场判断时 |
| 风险预算 | 按风险贡献配置 | 风险控制优先 |
| 目标日期 | 随时间自动调整股债比 | 长期理财 |
| 目标风险 | 维持固定波动率 | 风险厌恶型 |
| 因子暴露优化 | 控制Barra因子敞口 | 量化对冲 |

#### 3.2 约束条件扩充
**新增约束**：
- 行业偏离约束(相对基准±5%)
- 风格因子约束(Size/Value/Growth)
- ESG评分约束
- 流动性约束(最小市值/日均成交额)
- 个股集中度约束(前5大权重和<50%)

### Phase 5: 数据基础设施 (5-7天)

#### 5.1 数据质量保障
**问题**：
- 停牌、除权数据不准确
- 缺失数据影响回测

**解决方案**：
```python
# 数据清洗流程
1. 多源数据交叉验证
   - baostock (主要)
   - akshare (备份)
   - tushare (补充)

2. 异常检测
   - 价格跳空检测
   - 成交量异常检测
   - 收益率异常检测

3. 缺失值处理
   - 前向填充
   - 线性插值
   - 标记为NaN

4. 复权处理
   - 前复权(用于分析)
   - 后复权(用于回测)
   - 不复权(用于交易)
```

#### 5.2 数据缓存系统
**架构**：
```
Redis 缓存层
├── 热点数据(最近1年)
│   ├── 股票日线
│   └── 指数日线
├── 实时数据(5分钟过期)
│   ├── 实时行情
│   └── 资金流向
└── 计算结果缓存
    ├── 因子值
    └── 回测结果
```

#### 5.3 数据更新机制
**定时任务**：
```python
# 每日15:30更新
tasks.schedule.every().day.at("15:30").do(update_daily_data)

# 更新内容
1. 日线数据
2. 财务数据(季报期)
3. 指数成分股调整
4. 因子重新计算
```

## 五、关键决策点

### 1. 数据源选择
**方案A**：Akshare (免费，稳定性一般)

**建议**：复杂计算(CVXPY优化)放后端，简单计算放前端

### 3. 回测引擎
**方案A**：自研 (灵活，工作量大)

**建议**：自研核心引擎，参考Backtrader设计
