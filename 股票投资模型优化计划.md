# 股票投资模型专业版优化计划

## 一、现状分析

### 1.1 当前股票模型 vs 基金模型差距

| 模块 | 基金模型 | 股票模型 | 差距 |
|------|---------|---------|------|
| 筛选 | ✅ 完整功能 | ⚠️ 基础版 | 需要大幅增强 |
| 分析 | ✅ 完整功能 | ⚠️ 基础版 | 需要增强 |
| 组合 | ✅ 完整功能 | ⚠️ 基础版 | 需要开发高级功能 |
| 回测 | ✅ 8种策略 | ❌ 未开发 | 需要全新开发 |

打造与聚宽、米筐等专业量化平台看齐的股票量化投资系统。

## 三、详细优化方案

### 3.1 多因子选股系统

#### 目标：专业级多因子选股平台

**因子库设计（7大类40+因子）**

```
估值因子: PE(TTM)、PB、PS、PCF、股息率、EV/EBITDA
动量因子: 5/10/20/60日收益率、波动率、换手率、Beta
质量因子: ROE、ROA、毛利率、净利率、ROE稳定性
成长因子: 营收增长率、利润增长率、3年复合增长率
流动性因子: 总市值、流通市值、换手率
杠杆因子: 资产负债率、流动比率、速动比率
技术因子: RSI、MACD、布林带
```

**功能清单**
- [ ] 因子分类展示（折叠面板）
- [ ] 双滑块范围选择器
- [ ] 快捷筛选预设（价值、成长、质量、动量等）
- [ ] 策略保存/加载
- [ ] 分页排序
- [ ] 股票加入备选池

**后端API设计**
```
GET  /api/stock/factors          - 获取因子定义
GET  /api/stock/quick-filters   - 获取快捷筛选
POST /api/stock/screen          - 执行筛选
GET  /api/stock/strategies      - 获取策略列表
POST /api/stock/strategies      - 保存策略
DELETE /api/stock/strategies/{id} - 删除策略
```

## 七、注意事项

1. **不删除现有数据**：任何数据库操作不得使用 drop_all()
2. **向后兼容**：新功能不影响现有功能
3. **错误处理**：所有API需要有完善的错误处理
4. **性能优化**：大数据量需要分页和缓存
